# Analyze web app [https://base-connect-playground.vercel.app/](https://base-connect-playground.vercel.app/)

Suggest how to improve it's UI/UX and make not obvious that it was created with AI

You can make the Base Connect Playground feel more professional and less AI-generated by tightening the layout around the core wallet-connect flow, enforcing a consistent design system (spacing, colors, typography), and simplifying visuals and copy so everything serves that single task. Treat it like a small developer tool: one clear primary action (connect), high-signal logs, and minimal decoration, instead of a generic “AI app” landing page.[^1][^2][^3][^4][^5]

## Focus the layout on the connect flow

- Structure the page as: header → action column (connect controls) → state/logs, so the user always knows where to look and what to do next.[^2][^6]
- In the main area, keep a two-column layout on desktop: left for controls (connect, disconnect, network selector, options), right for current status and recent events; push long JSON/logs into a dedicated panel or tabbed area below.[^3][^2]
- Use clear empty states, e.g., “No wallet connected” with a short explanation, instead of blank panels or generic placeholders. This makes the tool feel like a purposeful SDK playground rather than a generic AI chat shell.[^4][^1]


## Enforce a simple design system

- Pick a small set of design tokens (colors, radii, spacing, font sizes) and apply them everywhere instead of mixing defaults the AI chose. For example, one primary Base-blue, one accent, and neutral grays for surfaces and borders.[^5]
- Standardize radii and spacing (e.g., 8 px grid) so buttons, cards, and inputs look like they come from one component library, not several stitched together.[^1][^5]
- Use one font stack and a clear hierarchy: H1/H2 for titles, strong label text for field titles, and monospace only in code/log areas; avoid random bolding and multiple font sizes for similar elements. This is how mature playgrounds (Vercel, Google, etc.) keep complex UIs feeling calm.[^2][^3]


## Make copy specific and developer-focused

- Rename any generic AI-style headings (“Welcome to your AI app”, “Playground for your ideas”) to something specific like “Base Connect Playground” with a short subtitle: “Test wallet connections and events on Base”.[^1]
- Use precise button labels (“Connect wallet”, “Disconnect”, “Switch to Base Sepolia”) and avoid vague CTAs like “Start” or “Try now”. That instantly makes it feel like a dev tool, not marketing.[^4][^1]
- Write one-sentence helper text under advanced options or parameters, and add meaningful empty-state copy in log/status panels instead of lorem ipsum or buzzwords.[^1]


## Improve feedback, state, and logs

- Add clear visual states on connect actions: loading spinner in the button, disabled state when an action is in progress, success state with a short confirmation and connection pill (e.g., “Connected · Base Sepolia”).[^4][^1]
- Surface important events (connect, disconnect, errors, chain mismatch) as small toasts or inline alerts with a link like “View in logs” rather than only as console-like text at the bottom.[^3][^1]
- Give the log area structure: tabs or filters for `connection`, `accountsChanged`, `chainChanged`, and `errors`, JSON formatted in monospace with syntax highlighting and copy buttons, and collapsible sections for long payloads.[^3][^4]


## Strip away “AI tells” in the visuals

- Remove decorative gradients, stock illustrations, or big hero sections that don’t serve the connect flow; SDK playgrounds typically look like tools, not marketing sites.[^2][^1]
- Align everything to a consistent grid, keep padding inside cards uniform, and avoid mixing pill buttons with sharp-edged cards or random shadows—this kind of inconsistency is a common AI artifact.[^5][^1]
- Eliminate any text that hints it was autogenerated (“AI-built UI”, template-y filler copy), and replace it with concise, product-specific descriptions and links to Base docs or the SDK GitHub.[^2][^4]


## Add small touches that signal craftsmanship

- Provide small but intentional navigation: a subtle top-right link group for “Base docs” and “SDK GitHub”, and a footer note like “Test-only playground for Base-compatible wallets · Built with Next.js on Vercel”.[^6][^4][^2]
- Ensure keyboard accessibility and good focus styles for buttons, tabs, and filters, and announce connection status changes to screen readers; accessibility polish is something AI-generated UIs often miss.[^6]
- Use light motion sparingly: consistent hover states and short transitions on status chips or log panel expansion, instead of default or exaggerated animations.[^1]
