# Multi-Chain Design System: One Product, Four Skins ## 1. Vision The goal is to create a singular "Playground" experience that feels native to four distinct blockchain ecosystems (Base, Celo, Optimism, Arbitrum) without fragmenting into four unrelated websites. We achieve this through a rigorous **Master Design Language** where layout, typography, and component shapes are identical, but the **Visual Skin** (color tokens and subtle graphic accents) pivots based on the active chain. ## 2. Master Foundations (Constant Across All Chains) - **Layout:** Two-column "Developer Console" layout. Navigation and primary interactions on the left/center, "Activity Log" terminal on the right. - **Typography:** Inter (San-serif) for all UI elements. Bold headers, mono-spaced font for terminal logs. - **Component Shapes:** Clean, sharp-cornered cards with subtle 8px border-radius. Minimalist flat buttons with distinct hover states. - **Spacing:** Tight, systematic grid (8px scale) for a dense, high-utility feel. - **Background:** Deep monochrome dark mode foundation (#050505 or #0E0E0E). ## 3. Chain-Specific Logic - **Base:** Restrained. Use blue only for the primary switcher state, the main "Run" action, and active status pips. - **Celo:** Warm. Use Prosperity Yellow for highlights. Incorporate soft "human" forest green tones in the terminal and background surfaces. - **Optimism:** Assertive. Use the red for editorial-style bold accents, border highlights, and critical "Live" indicators. - **Arbitrum:** Layered. Use navy backgrounds with teal highlights to create a sense of depth and technical precision.
